Sintese da aula de hiperlinks

Hiperlink ou link

Trata-se de uma referencia que está dentro de um documento em hipertexto que direciona para outras partes do site ou de outros sites, havendo tipos diferentes como

  1. Remotos
  2. Internos
  3. Locais
  4. Imagens
  5. Com indereço de correio
  6. Com indereço de arquivos

Sua marcação:

Acontece se utilizando a tag a e a propriedade href, responsavel por fazer a referencia para o conteúdo desejado..

Sintaxe


Imagem dos slides do professor Francis... Todo o crédito a ele..

Hiperlinks Remotos

São os links que fazem referencia a sites e paginas da web, sendo assim, em href, colocamos a URL do site

Temos duas propriedades legais para links remotos, a _blank que abre o site em outra guia da web e a _self que abre na mesma guia, as duas sao utilizadas na opção target

Hiperlinks Internos

São os links que fazem referencia a outra parte do mesmo site, podendo acessar outros paragrafos e seções..

Hiperlinks Locais

São os links que fazem referencia a outras paginas do mesmo site, tendo que colocar o nome do arquivo html dentro da tag href..

Hiperlinks em imagens

São os links que fazem referencia a outras paginas, juntando com a tag de imagem

Hiperlinks com indereço de correio

São os links que fazem referencia a um indereço eletronico, um exemplo esta disponivel ai embaixo direcionando para o meu email

caiorwendre@gmail.com

Hiperlinks com indereço de arquivos

São os links que fazem referencia a arquivos e podem fazer download dos mesmos, com as tags download e type

Software de Vercionamento

Trata-se de um software que cria um histórico de versões para um melhor controle das mesmas, sendo que se pode criar um repositório remoto que fica em um servidor e que pode ser acessado de dferentes lugares por muitas pessoas diferentes.

  • Tipos de Software de Controle de Versão
  • Git
  • Github
  • Preparando o ambiente

Tipos de Software de Controle de Versão

Centralizado

Possui um repositório central em um servidor com controle de versões.
É dependente do Servidor.

Distribuido

Possui um repositório local com controle de versões.
Atualizações no repositório central.
Independente do Servidor.

Git


Logo do Git

É um software livre de vercionamento que surgiu para o desenvolvimento do Kernel do Linux, o qual foi projetado e desenvolvido por Linus Tolvalds em 2005

Cada diretório de trabalho do Git é um repositório com um histórico completo e habilidade total de acompanhamento das revisões, não dependente de acesso a uma rede ou a um servidor central.

Logo do Git

Este site é uma Sintese da aula de Branches e Hospedagens

Branches são um ferramenta de versionamento, para adicionar ou retirar conteudos posteriores, sendo repartidas as versões, a versão original é particionada em branch Master ou main, e a parte a ser adicionada ou em fase experimental, é adicionada em uma branch secundaria, de forma que não haja perigo de comprometer a original, depois podemos criar o que queremos no VsCode e fazer um commit e dar um Push Origin para upar para a branch secundaria no site da github..

Criando e fazendo uma branch

Para criar uma branch no Desktop Github é bem simples, quando criamos um repositorio, ja vem a branch main junto, então, de cara estamos nela, entã para criar uma branch secundaria, ou terciaria, dependendo da utilidade, devemos ir na gua do programa que está escrito "Branch", logo após... Clicamos na opção "New Branch", nela damos o nome da branch e a parte que ela herdará, podendo ser da main ou de outras branchs criadas anteriormente...

Adicionando as alterações da branch secundaria a main:

Para fazer a junção da branch secundaria com a main, é necessario selecionar a branch main no Github Desktop, apos isso vamos na guia Branch e clicamos em "Merge into current branch..", abrirá uma janela onde selecionamos a branch que queremos unificar ou atribuir a main e clicamos no botão azul na parte inferior da janela, podendo aparecer conflitos de codigo que podem ser corrigidos... Após isso damos um Push to Origin para sincronizar com nosso repositório remoto, e assim atualizará as duas branchs..

Fazer um update direto pra main

Com a branch secundaria selecionada também temos a opção de fazer a atualização para a main direto dai, de forma a clicar na aba Branch, e em seguida, na opção "Update fron Main"

Hospedagem

Em geral para hospedarmos um site em um servidor é necessário pagar, porém, existe um recurso da GitHub que se chama GitHub Pages>, onde podemos criar uma conta em um repositorio e assim o repositorio pode virar um site de forma que é gerado um link da github com o nome do criador e o nome do repositorio.. Para fazer isso vamos na pigina do repositorio na github, vamos na opção do desenho das engrenagens e vamo até a parte de Github Pages, clicamos na caixinha e marcamos main, e damos um save, onde automaticamente sera gerado o link do site do repositorio..
