Este site é uma Sintese da aula de Branches e Hospedagens

Branches são um ferramenta de versionamento, para adicionar ou retirar conteudos posteriores, sendo repartidas as versões, a versão original é particionada em branch Master ou main, e a parte a ser adicionada ou em fase experimental, é adicionada em uma branch secundaria, de forma que não haja perigo de comprometer a original, depois podemos criar o que queremos no VsCode e fazer um commit e dar um Push Origin para upar para a branch secundaria no site da github..

Criando e fazendo uma branch

Para criar uma branch no Desktop Github é bem simples, quando criamos um repositorio, ja vem a branch main junto, então, de cara estamos nela, entã para criar uma branch secundaria, ou terciaria, dependendo da utilidade, devemos ir na gua do programa que está escrito "Branch", logo após... Clicamos na opção "New Branch", nela damos o nome da branch e a parte que ela herdará, podendo ser da main ou de outras branchs criadas anteriormente...

Adicionando as alterações da branch secundaria a main:

Para fazer a junção da branch secundaria com a main, é necessario selecionar a branch main no Github Desktop, apos isso vamos na guia Branch e clicamos em "Merge into current branch..", abrirá uma janela onde selecionamos a branch que queremos unificar ou atribuir a main e clicamos no botão azul na parte inferior da janela, podendo aparecer conflitos de codigo que podem ser corrigidos... Após isso damos um Push to Origin para sincronizar com nosso repositório remoto, e assim atualizará as duas branchs..

Fazer um update direto pra main

Com a branch secundaria selecionada também temos a opção de fazer a atualização para a main direto dai, de forma a clicar na aba Branch, e em seguida, na opção "Update fron Main"
